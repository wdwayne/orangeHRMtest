# OrangeHRM QA Testing Project

## 📌 Project Overview
This project showcases a complete manual QA testing process performed on the OrangeHRM application. It includes test planning, exploratory testing, test case design, and defect reporting.

The goal of this project is to demonstrate real-world QA skills in identifying defects, validating system functionality, and documenting testing activities.

---

## 🖥️ Application Under Test
- Application: OrangeHRM (Demo)
- Type: Web-based Human Resource Management System (HRMS)

---

## 🎯 Testing Scope

### In Scope
- Login functionality
- Navigation modules
- Employee management (PIM)
- Dashboard features

### Out of Scope
- Third-party integrations
- Backend/database validation
- Performance testing

---

## 🧪 Testing Types Performed

- Manual Testing
- Functional Testing
- Exploratory Testing
- Negative Testing

---

## 📂 Test Artifacts

This project includes the following QA deliverables:

- 📄 Test Plan  
- 🔍 Exploratory Testing Notes  
- ✅ Test Cases (Login & PIM Modules)  
- 🐞 Bug Reports  
- 📊 Test Execution Report  

---

## 🐞 Key Findings

During testing, the following issues were identified:

- **BUG-001:** Incorrect error message displayed ("CSRF token validation failed") instead of user-friendly authentication error  
- **BUG-002:** Dashboard accessible after logout via browser back button (session handling issue)  
- **BUG-003:** Inconsistent search results in PIM module under certain conditions (under investigation)
